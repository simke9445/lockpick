Lockpick agent guide

Inspect:
- lockpick capabilities --json
- lockpick doctor --json
- lockpick status --json
- lockpick identify --json

Lock before edits:
- lockpick acquire <paths...> --reason "<intent>" --id-only
- lockpick expand --lock <lock_id> <paths...>
- lockpick refresh <lock_id>

Commit with the shared Git index:
- lockpick git begin --refresh-lock <lock_id> --reason "<commit intent>" --id-only
- git add <locked_paths>
- git commit
- lockpick git end <git_lock_id> --release-lock <lock_id>

Recover:
- If a conflict returns prune_then_retry, run lockpick prune --dry-run --json, then lockpick prune.
- If an unknown flag error prints next:, run that exact command.
- If ownership fails, retry with the owning --owner-session or work on unlocked files.

Output contract:
- Prefer --json for data surfaces.
- Treat stdout as data and stderr as diagnostics.
- Read exit codes from lockpick capabilities --json.
